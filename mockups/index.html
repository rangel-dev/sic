4 telas redesenhadas

1 · Home / Dashboard

KPIs reais com sparklines, delta vs ciclo anterior, operações em andamento e timeline com outcomes.

2 · ProcessPage (Conversor)

Componente reusável: stepper visual, drop zones modernas, log lateral em tempo real, cards diferenciados por ícone.

3 · Tabela de resultados

Filtros inline, ordenação, ações em massa, empty state ilustrado, paginação.

4 · Command Palette (Ctrl+K)

Fuzzy search de módulos e ações, atalhos visíveis, recentes — produtividade para usuários power.

Como ler os mockups

Tudo aqui é apenas HTML/CSS estático. As interações principais (hover em cards, busca no command palette, filtros na tabela) funcionam só visualmente — não há lógica real conectada. O objetivo é validar a direção visual antes de gastar horas reimplementando em PySide6/QSS.

Cada cor, raio de borda, espaçamento e fonte usado aqui já existe no qss_dark.py — a tradução para Qt é direta.
